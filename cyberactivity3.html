    SolarWinds Exploit Analysis

    Phase Description Identification Status
    Reconnaissance The attackers likely conducted extensive reconnaissance to identify potential targets and gather information. Identified
    Weaponization The attackers weaponized the SolarWinds software update with a malicious backdoor (SUNBURST) before distribution. Identified
    Delivery The weaponized update was delivered to SolarWinds' customers through legitimate software update channels. Identified
    Exploitation Once the malicious update was installed, the backdoor exploited vulnerabilities and established a foothold. Identified
    Installation The attackers established persistence on the compromised systems using the backdoor. Identified
    Command and Control, C2 OThe attackers set up communication channels to remotely control the compromised systems Identified
    Actions on Objectives The ultimate goals of the attackers may have included data exfiltration, further lateral movement, etc. Identified

    Create a list of possible mitigations for each phase. Are there any phases you cannot mitigate?

    1. Reconnaissance:
      • Implement strict access controls on sensitive information.
      • Monitor and analyze external information requests.
      • Educate employees about the risks of oversharing information online.
    2. Weaponization:
      • Employ email filtering to detect and block malicious attachments.
      • Keep software and applications up-to-date with the latest security patches.
      • Implement endpoint protection solutions to detect and prevent malware.
    3. Delivery:
      • Employ email security solutions to detect and block phishing emails.
      • Use content filtering to block malicious websites.
      • Implement multi-factor authentication (MFA) for critical systems.
    4. Exploitation:
      • Regularly patch and update software to address known vulnerabilities.
      • Employ intrusion detection/prevention systems (IDS/IPS) to detect and block suspicious activities.
      • Utilize application whitelisting to allow only approved programs to run.
    5. Installation:
      • Implement robust endpoint security solutions to detect and remove malware.
      • Conduct regular security audits and penetration testing.
      • Employ host-based firewalls to limit inbound and outbound traffic.
    6. Command and Control (C2):
      • Utilize network monitoring tools to detect anomalous traffic patterns.
      • Implement network segmentation to limit lateral movement.
      • Employ threat intelligence feeds to identify known C2 servers.
    7. Actions on Objectives:
      • Implement data loss prevention (DLP) solutions to monitor and prevent unauthorized data exfiltration.
      • Conduct regular security awareness training for employees to recognize and report suspicious activity.
      • Employ advanced threat hunting techniques to proactively identify and respond to threats.

    What tools would you utilise in each phase? Give reasons for your answer.

    1. Reconnaissance:
      • Shodan and Censys: These are search engines for internet-connected devices. They can be used to identify potential targets by searching for specific software or services that may be vulnerable.
    2. Weaponization:
      • Metasploit: A widely-used penetration testing framework that can be used to create, test, and execute exploits. In the SolarWinds exploit, attackers weaponized a legitimate software update, so specialized tools were not necessary for this phase.
    3. Delivery:
      • Email Security Gateways (e.g., Proofpoint, Mimecast): These solutions filter inbound emails for malicious content and attachments. They can detect and prevent phishing emails, including those delivering weaponized payloads.
    4. Exploitation:
      • Exploit Kits (e.g., Angler, Rig): In the case of a different type of exploit, exploit kits can be used to deliver and execute malicious payloads on vulnerable systems. However, in the SolarWinds exploit, the exploitation occurred through the compromised software update, making the use of specialized exploitation tools unnecessary.
    5. Installation:
      • Endpoint Protection Platforms (EPP): Solutions like CrowdStrike, Symantec, and others are used to protect endpoints from malware and other security threats. They can detect and remove malicious software, including the backdoor installed in the SolarWinds exploit.
    6. Command and Control (C2):
      • Network Monitoring Tools (e.g., Wireshark, Suricata): These tools can be used to monitor network traffic for unusual patterns or connections to known malicious domains. This can help identify and block communication channels established by the attacker.
    7. Actions on Objectives:
      • Data Loss Prevention (DLP) solutions (e.g., Symantec DLP, McAfee DLP): DLP tools monitor and prevent unauthorized data exfiltration. They can be configured to detect and block the unauthorized transfer of sensitive information.

    Primarily, executives must recognise cyber security as a crucial component of their enterprise, akin to ensuring that their workplace is secured after hours (Almeida et al., 2020; Spremić & Šimunic, 2018). Because they haven't previously been exposed to cyber security threats, industries that are new to digitalization face unknown challenges (Almeida et al., 2020). Hackers discovered new vulnerabilities in cars as they integrated more digital components, endangering the integrity of those systems (Spremić & Šimunic, 2018). A McKinsey and Company study estimates that the majority of contemporary cars contain 100 million lines of code; in the next ten years, this number is predicted to triple (Deichmann et al., 2019).
